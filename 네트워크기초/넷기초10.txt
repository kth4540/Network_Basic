 # 서킷 - 스위치 캐리어 서비스 (서킷 네트워크)
 : 독점적으로 나 혼자 쓰는 것처럼 사용(전용회선), 내가 사용하지 않는 순간에도 다른 사람이    쓸 수 없음. 음성정보나 텍스트를 주고 받음. '콜 셋업'이라는 사전시간이 필요함. 서로 수화    기를 들고 통화 할 준비 되도록 해야함. 요금을 시간에 비례해서 부과함. A에서 B까지 서로     를 통해서 회선이 연결되어 이동함.
 
 # 패킷 - 스위치 캐리어 서비스 (패킷 네트워크)
 : 독점적인 회선이 아님. 다른 사람들도 사용이 가능하다. '콜 셋업'이 필요하지 않음.
   연결회선이 낭비되지 않는다. 다른 사람의 것을 써서 보낼 수 있기 때문이다.
   라우터가 어느 방향으로 가는 것이 좋을지 결정을 내려줌. 서킷에 비해 네트워크 사용도가
   크다. 회선전체를 타고 이동하는 데이터양이 서로 비슷하다. 덜 몰리는 곳에는 데이터를
   더 보내 비슷비슷하게 만들어 효율적인 사용을 한다.

 * 통신 사업자입장에서는 패킷이 망관리가 더 쉬우나, 개별 이용자 입장에서는 속도가 느릴        수 있다. 그렇기에 요금이 더 싸다.(정액제방식)

 * ISDN : 아날로그 방식으로 제공하던 PSDN을 디지털 방식으로 바꾼 것.
 * SONET : 광케이블방식으로 구축.

 * DSL : 전화와 인터넷을 전화선으로 동시에 쓰지 못했음. 가능하게 해준 것이 DSL이다.
            라인 스플리터와 컴퓨터 사이에 DSL 모뎀으로 나누어서 연결. 로컬 루프에 연결된
            CO에서는 목소리와 데이터를 구분함. 큰 돈이 들지 않고 관리가 편함. 추가 장비로
            기존것들을 그대로 쓰면서 사용이 가능함.
 - ADSL(비대칭) : 업로드에 비해 다운로드가 더 많으므로 회선의 폭을 다르게 함.
	           다운로드 속도가 빨라짐.(가변차선제)
 - VDSL : ADSL보다 속도를 높인 버젼(기능은 같음)
 - FTTH : 100메가 이상의 속도.

 * CATV : 케이블 방송 사업. CO(케이블 방송 관리자)/DSL과 비슷한 방식으로 스플리터를
	통해서 TV와 컴퓨터를 구분함. 로컬 루프이후의 CMTS에서 구분해줌.
 * HFC(Hybrid Fiver Coaxial) : 동축케이블과 광케이블을 함께 사용.
 * PON : Ethernet기반인 EPONs를 우리나라에서 사용 -> FTTH

 #Wireless : WiMAX -> WiBro , 와이맥스를 우리의 기술로 개량한 것이 와이브로.(망함)
 * VoiceNW : 유선전화 - 무선전화 - 휴대전화
 * DataNW : 유선인터넷 - 무선인터넷(WiFi) - 휴대인터넷(WiBro)
 - 1997년부터(2세대) 전화로도 텍스트 데이터를 주고 받을 수 있게 됨. 텍스트->동영상...
   2~3G 때 LTE연구가 이뤄지고 있었음. 또 이용패턴이 노트북에서 휴대폰으로 이동했기에 
    WiBro보다 LTE가 더 유리했다. VoIP서비스가 WiBro가 있었으면 유리했을 수도 있다.
 * 멀티플렉서 : 한 쪽에서는 신호를 묶고 반대쪽에서는 신호를 풀음. 묶는 방법은 여러가지가
    존재한다. 신호를 한번에 묶어서 보내면 원하는 신호만 뽑아내 이용가능하다.
