 
 * LAN ARCHITECTURES : 전송 매체로 전송 시에 지켜야 하는 규칙
 - ETHERNET : 아키텍쳐의 대표적인 예.알로하넷을 이용해 데이터를 전송.
	        알로하넷 : 하와이 네개의 섬을 연결하려고 만든 네트워크, 전송규약 : 이더넷
	        DIX라는 이름으로 처음에 규정되었다. 싸고 좋다. 현재 가장 각광을 받음.

 - 10Base5 : 10메가비트의 속도를 내며, THINKNET을 사용
 - 10Base2 : 10메가비트의 속도를 내며, THINNET을 사용
 - 10Base-T : 10메가비트의 속도를 내며, Ethernet UTP를 사용
	- 뒤의 숫자는 전송거리를 반올림한것이다.(앞 : 속도, 뒤 : 케이블의 종류)

 * ACCESS METHOD : 전송방법(CSMA/CD)
 - Carrier sense : 전류의 크기를 재서 망을 제대로 통과하고 있는지 확인(Sensing)
	           이동데이터의 유무를 판단하여 내 데이터를 내보냄.(가끔 충돌존재)
 - Mutiple access : 모든 데이터들이 망에 접근할 수 있는 동등한 권리를 가짐. 우선순위X
 - Collision detection : 간발의 차이로 충돌이 생기는 경우 충돌을 찾아냄. 충돌 시,
		   랜덤한 시간을 기다렸다가 다시 보냄. 두 컴퓨터의 충돌 시, 서로 다른
		   랜덤시간을 기다렸다가 보냄.
 - CSMA/CD : 로직이 쉽다. 필요한 장비도 적다.(장점)
	       Ethernet에 연결된 컴퓨터가 많아지면 충돌가능성이 높아짐.딜레이 발생(약점)
	       그래서 Ethernet은 거리제한을 두고 컴퓨터의 수도 제한을 둠.
