 #LAN ARCHITECTURES - WIRELESS : 근거리에서 작동
	- 비용(아직까진 같은 비용대비 유선보다 못하다)
	- 서비스 할 영역에 따라서 적절한 아키텍쳐의 고려가 필요
 * IEEE 802.11(표준)/IEEE 802.15(블루투스) : 지속적으로 업데이트, 새로운 버전 넘버링
 
 - IEEE 802.11 : DCF를 이용하여 충돌을 피하는 기술(collsion avoidance)
 - Bluetooth : polling mechanism(master가 slave에게 한번에 하나씩 물어보는 것)
	     piconet을 사용(장:매우쉽게충돌구현/단:slave가 많아지면 기다리는 시간증가)
 - HomeRF(라디오주파수) : 집안의 가전기기 등을 무선으로 커뮤니케이션
		           TDMA - 기술은 복잡하나 원활하게 전달 가능
 * Multipelxing(다중화) : 하나의 주파수에 다른 데이터를 섞어서 보내는 것, 1대1통신은 가능0
 * FDDI : 과거에 각광을 받음.광케이블사용.각각의 건문들이 하나로 연결됨(듀얼링 토폴로지)
	이제는 이더넷을 사용.
 * ATM : 60년대 후반에 벨랩에서 개발.음성을 제외한 다른 데이터 전송을 목적으로 개발됨.
	초고속, 지연의 최소화.최근에 와서는 기가빗이더넷이 적용됨.

 #LAN DEVICES
 * Repeater(증폭기) - 신호가 전송되는 거리를 늘려줌. 다음 전송매체로 신호를 전달.(단독X)
		   허브나 스위치안에 존재==허브나 스위치는 증폭기능이 존재
		   ex)thinet의 경우에는 최대 185m당 배치해야 함.
 * Hubs - 스타토폴로지 사용 중, 허브&스위치 사용.모든 신호를 다뿌림(broadcast - 1:n(all))
	연산기능이 없다(지능이 낮다), 들어온 데이터를 분석하지 않음.
	! unicast -> 1:1/multicast -> 1:n(all이 아닌 some)
 * Bridge - MAC address(2) 사용 장비. 중간에 연결해줌.
 * Switches - 브릿지와 비슷.MAC address를 이용해서 찾아감. 브릿지는 허브에만 작용하나,
	     스위치는 여러곳에 작용가능. 동시에 여러곳 처리가능(스위치>브릿지)
 * Routers - 길을 찾아주는 것.네트워크와 네트워크를 연결.목적까지 가는 최적의 경로를
	    알아내고 그 경로로 데이터를 이동시켜 줌. IP 주소를 통해 식별하여 경로를 찾음.
	    3계층디바이스이다.브로드캐스트 메세지를 걸러냄.
 * Gateways - 요즘에는 잘 쓰이지 않음.네트워크와 네트워크를 이어주는데 쓰이는 연결장치
	       네트워크 사이에 사용하는 프로토콜이 서로 다른 경우 사용(번역기능이 존재)
	       
 
 * 데이터네트워크==패킷네트워크(이동단위가 패킷단위)
 * PAD : 패킷전송 시에 나눠서 전송하고 받을 때 하나로 합쳐서 순서대로 정리해서 모으는 기능
