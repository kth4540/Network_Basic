# 데이터 커뮤니케이션=네트워크(데이터를 주고받는 서비스 총칭)
		A -> B (하나씩의 전송매체(봉화라면 공기)를 가짐)

 - 데이터 커뮤니케이션은 언제부터 있었는가? -> 예전부터...
	ex) 봉화(연기를 피우는 시스템), 마라톤의 기원(내용 전달)
	     우편(내용을 써서 전달), 전서구(새를 통해서), 휴대폰
 - 과거에 비해 종류가 다양해지고, 속도가 빨라졌다.
 - 전송매체를 가지고, 이동전에 포맷의 과정(데이터의 전기신호를 변경)을 거침.
 - 최근에는 하이테크의 커뮤니케이션이 사용 : 하드웨어, 소프트웨어 등등.
 - 두개 이상의 점, 사람들, 기관 사이에 전송매체에 특화된 형태로 포맷이 되어
    인코딩 된 데이터와 정보의 이동을 의미한다.

# 데이터 인코딩 방식

 - EBCDIC : IBM에서 만든 메인컴퓨터 인코딩 방식.
 - ASCII : PC를 타겟으로 만든 미국 표준 인코딩 방식.(영어, 숫자, 특수문자 1byte)
	2의 8승 = 256가지
 - Unicode : 한글의 경우 초성, 중성, 종성의 사용으로 256가지로는 표현 불가.
	2의 16승의 서로 다른 캐릭터(우리나라포함 256가지로 불가능한 언어들)


# 데이터 이동방식	Analog(0~1그 이상)		vs	Digital(오직 0과 1)
 - 데이터는 이동 중 손실이나 잡음으로 전달이 완벽하지 않음.(찌그러진 데이터)
 - 도중에 데이터를 다시 원래 데이터로 회복함.
	디지털 : 데이터를 반올림하여 원래 데이터를 회복.(완벽한 복원)
	아날로그 : 데이터를 똑같은 비율로 잡아늘임.(구간마다 달라 완벽한 복원불가)
 	* 복원력이 더 좋은 디지털이 데이터 전달이 깨끗하다.(디지털>>>아날로그)

#  Parallel Transmission(병렬통신)	VS	Serial Transmission(직렬통신)
	프린터기와 컴퓨터				USB		

Synchronous Transmission(동기통신/한번에 준비하여 정보를 보냄.보내는 쪽 받는 쪽 동시)
Asynchronous Transmisson(비동기통신/중간에 stopbit를 두어 보냄.데이터 많으면 불리)

# Simplex, Half-Duplex, and Full-Duplex Transmisson
	simplex : 한 쪽 통신(터치 기능이 없는 모니터)
	Half-Duplex : 양 쪽 통신은 가능하나 한번에 한 쪽만 가능(무전기)
	Full-Duplex : 양 쪽으로 동시간에 통신가능(휴대전화)
